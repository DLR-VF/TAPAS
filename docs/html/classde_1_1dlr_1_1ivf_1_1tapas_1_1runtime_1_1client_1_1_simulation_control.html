Detailed Description

The SimulationControl represents the controller for the simulation client. The controller has three active tasks and provides methods to change the simulation entries in the database.

The active tasks are:

  1. SimulationDataUpdateTask
    This task updates the simulation values in the memory of the client from the database
  2. SimulationServerDataUpdateTask
    This task updates the simulation server data values in the memory of the client from the database
  3. TimeUpdateTask
    This task retrieves the current timestamp of the database

    Furthermore the client provides methods to add or remove simulations from the database and to change the state of a simulation, e.g. it is possible the stop or restart a simulation.

void de.dlr.ivf.tapas.runtime.client.SimulationControl.changeSimulationDataState ( String  sim_key)

This method changes the state of the simulation corresponding to the parameter sim_key.
The changes are:

INSERTED -> STARTED
STOPPED -> STARTED

STARTED -> STOPPED
FINISHED -> Remove simulation completely from the database

The changes are done via a SQl statement and the skipUpdate flag is set to true after the statement was committed.

sim_keykey for the simulation in the database
